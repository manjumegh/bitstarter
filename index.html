About

"Do Charity, and Count Your Blessings"
Charity-Box is social networking site with the cause. This is little effort to reduce the gap between poor and rich people. By providing financial/social help on right time (when there is need) and from right people (connected, affordable people) for right people (authenticated individual using social networking strength). The needy user (poors or victims of riots/attacks/natural calamities (earthquake/flood/tsunami)) or their friends or relatives will post need request through Charity-Box. The request will be broadcasted to all, with posting user's relation. The Charity-Box through social networking, provides help by reaching to right person. In return of charity, the donors will recieves Charity-Points, equal to donation/charity made by an individual. The Charity-Points identifies value of an individual in Charity-Box. Charity-Points are measure of an individuals' donation. With every charity (intellectual/material), the reciever will give charity points to donar. More charity makes an individual more important. This way the people in need whether an individual himself or individual's friend/relative, get authentication of "legal need" inherently by using social-networking strength. The Charity-Points are transferrable from one individual to other individual in network.
This can be considered as bridge to decrease the gap of poor (demand) and rich (supply). How is it different from other social networking site, is explained with example in FAQ.

FAQ

How is Charity-Box different from other Social networking sites ?
Charity-Box is social networking site with the cause. Its an effort to reduce the increasing gap between rich and poor people. It achieves this by providing the help at right time (using networking) to right people (authenticated needy people social network) from right people (an affordable known people). This way the people in need whether an individual himself or individual's friend/relative, get authentication of "legal" inherently by using social-networking strength. Here, we will refer request for charity as demand and response to that request as supply. As shown in Figure, e.g., An individual's friend's father met an accident details attached, needs kidney for his surgery. He will raise request, call it as demand, as shown in figure. The response is mentioned as supply. If we search/raise request on
  • Google search engine, it will display all the results, shown in bigger circle, with blue dots, which are either links of organ banks, or organ donar. The google search results are not satisfactory because, it doesn't give relevant results. It lacks authentication of an individual. It shows results of individual who have posted on the web or have independent website.
  • Facebook/Google+, as shown in figure, it shows response of an individual friend's circle/group. This may or may not provide the solution to raised request.The results' reach-ability is as close as asking individual's friend. It provides authentication, but search space is very small. Thus, response space is very limited.
  • Twitter, as shown in figure, when the request is tweeted, that will be visible to an individual's follower. These followers may or may not be individual's friend. This is shown in figure, using overlapping circles between, twitter and facebook. The result lacks the authentication and has very limited search space. The space is confine to only followers.
  • In Charity-box, It starts from an individual's friend circle, and using social networking strength it can grow to an anonymous individual. The anonymous individual may not be known person, but it can provide response to the raised request. Here, the search space is not limited, it is growing unless response is provided. It provides inherent authentication from an individual.
The Charity-Box is dynamically growing social networking site, till the response of raised request is met. It is elastic social-network. An individual network grows temporary, till it gets response of raised request. It comes back to same size once request is responded, thus individual network is elastic in nature. The growing network is shown in figure with "dotted" circles.
The comparison is as shown in figure,the dots represents search results. When requested queries are serached using different social networks are shown. Google search results are bigger but the result is irrelavant and doesn't serve the request. Facebook Twitter and Google+ search space is very limited, confine to either friends or followers, but the results are authenticated, due to social networking strength. Whereas, "dotted red-color", Charity-Box, search space is elastic and authenticated. This grows temporarily till raised request is responded.
How does it work?
Its like any other social network site, every new individual opens account, with its mail id, user name and password. Once the account is created, the profile is created using:
  • Personal Information: Email Id, contact/mobile number.
  • Professional Information: Current profession, past profession, education and all the information from linkedin.
  • Security Settings: Password, authentication question.
  • Friend List: Populate friend list from, gmail id, facebook contacts, linkedin contacts.
Each user creates profile and that profile will be used for social networking. An individual, post request, and request is responded in dynamically growing network by another individual, as shown in above figure. The individual can post request and respond to posted request. Each raised request consists of:
  • Request Statement: One line or few lines request statement. It can be demand based request, in which each individual raise request if there is demand/need of charity.e.g.
    • "Mr. XYZ met with an accident need immediate blood (O -ve) and $10,000 for brain surgery."
    • "Needs volunteer for blind school primary class teaching for 5 hrs/month"
    • "I need guidance for taking admission in university ABC for XYZ course"
    • "Tsunami Relied Funds, donate old cloths, moey etc."
    It can be supply request: In which an individual have enough resources (money/time/energy) to donate. An individual wants to know the proper (authenticated) channel/medium and needy person to get benefit from that donation. Example of request can be:
    • "I want to donate old books, toys and school accessories (pencil/pen/notebook)"
    • "I want to donate $10/month for cancer treatment or cardiac treatment"
  • Supporting Attachment: Each raised request (demand request) will have supporting documents. Example, Doctors' prescription, hospital's admission form, any authenticating/supporting documents.
  • User options: Each raised request will have user's option e.g. demand request have , "I Know", "I Know and I donated", "Share", whenever an individual come across the raised request, it will pass that request to his/her contacts, with following options. Each of these options are counters, if an individual knows raised request, he/she will click on "I Know" option, each click increases counter. This way it will get inherently authenticated by individuals, who views the raised request. Increased counter indicates more authenticated request. It keeps increasing untill counter "I Know and I donated", can make the requested amount "0". If an individual doesn't know ab't "request statement", it will simply click on "share" counter, for helping to find donar. Once the reciever gets donation the auto-acknowledgement is sent to charity-box. The charity-box calculates equivalent charity-points corresponding to received auto-ack. Those charity-points will be sent to donar accounts.
